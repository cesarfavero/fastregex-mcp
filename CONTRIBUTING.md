# Contributing

Obrigado por contribuir com o fastregex-mcp.

## Regras gerais

- Este projeto é **liderado e aprovado por Cesar Favero**. Toda mudança depende de revisão e merge manual.
- Abra um **issue** antes de PRs grandes ou mudanças de arquitetura.
- PRs devem ser **pequenos e focados**.
- Não faça mudanças de branding, créditos ou autoria sem aprovação explícita.
- Não altere APIs públicas sem discussão prévia.
- Não inclua benchmarks novos sem **scripts e resultados reproduzíveis**.

## Checklist de PR

- [ ] Explica o problema ou motivação (link para issue, quando existir)
- [ ] Mudanças pequenas e isoladas
- [ ] Testes rodados (ou justificativa)
- [ ] Documentação atualizada quando necessário

## Processo sugerido

1. Abra um issue descrevendo a proposta.
2. Espere feedback/aceite.
3. Abra um PR pequeno, com descrição clara.
4. Aguarde revisão.

## Estilo técnico

- Preferir mudanças determinísticas e mensuráveis.
- Evitar otimizações sem benchmark comparativo.
- Preservar compatibilidade da API MCP.
